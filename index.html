핵심 요구사항 분석

  • 주문 정보 관리 — 하우저 시스템 API 연동으로 실시간 주문 데이터 및 배송 상태 조회, 배송 D-Day 추적
  • 고객 상담 이력 관리 — 채널톡 Open API 연동으로 상담 내역 동기화, 상담 메모 작성, 상담 분류(불만/일반문의/상담중) 색상 코딩
  • 관리자/사용자 관리 — 10~20명 사내 임직원 RBAC 3종 역할(슈퍼관리자, CS관리자, CS상담사)
  • UI/UX 디자인 전담 — 기획~디자인~개발~배포 올인원 수행 (클라이언트 디자인 인력 미보유)
  • 기획 산출물 — 요구사항 정의서, 기능 정의서, 화면 설계서 3종 납품
  • 상담사 퍼포먼스 대시보드 — 응대 건수, 처리 시간, 상태별 통계 시각화
  • 참고 이미지 분석 반영 — VIP 고객 태깅, SMS/알림톡 발송, 엑셀 다운로드, 통합 검색 기능 확인

제안 포인트

  1. 채널톡 + 하우저 이중 API 연동 — 상담 이력과 주문/배송 상태를 단일 고객 프로필에서 통합 조회. 참고 이미지 분석 결과, 고객 사이드바에 프로필·주문·상담 탭을 통합하는 UI 패턴 확인
  2. RBAC 기반 3종 역할 관리 — EZ-Approve에서 7종 역할 관리 경험을 3종(슈퍼관리자·CS관리자·CS상담사)에 맞춤 적용
  3. 상담사 퍼포먼스 대시보드 — 참고 이미지의 "통합 상담 이력 조회" 화면을 기반으로 필터링, 통계, 엑셀 다운로드 기능 구현
  4. 올인원 수행 — 기획 문서 3종 + Figma 디자인 + 풀스택 개발 + 배포까지 단일 팀에서 일관성 있게 진행
  5. 유사 프로젝트 코드 재활용 — EZ-Approve(RBAC, API 연동), Harmony Link(고객 이력 통합 관리) 아키텍처를 직접 재활용하여 효율적으로 개발

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

CS 인입 고객을 효율적으로 응대하고 관리하기 위해 전용 CRM 시스템을 구축합니다. 현재 채널톡과 배송업체(하우저) 시스템이 분리되어 있어 상담사가 고객 정보를 파악하기 위해 여러 시스템을 전환해야 하는 비효율이 발생하고 있습니다.

목적:

  • 채널톡 및 하우저 시스템 연동을 통한 고객 정보 통합 뷰 제공
  • 상담 이력과 주문/배송 상태를 한 화면에서 확인 가능한 환경 구축
  • 상담사 퍼포먼스를 한눈에 파악할 수 있는 대시보드 제공
  • 10~20명 규모의 사내 임직원이 사용할 수 있는 안정적인 시스템 구축

2. 사용자 정의

슈퍼 관리자시스템 전체 관리, 설정 변경 권한사용자 관리, 권한 설정, 시스템 설정, 전체 대시보드, 엑셀 다운로드
CS 관리자CS 팀 리더, 상담사 관리상담사 퍼포먼스 조회, 상담 배정, 통계 대시보드, 통합 상담 이력 조회
CS 상담사고객 응대 담당자고객 조회, 상담 이력 확인, 메모 작성, 주문 조회, 알림톡 발송

3. 기능 요구사항

3.1 주문 정보 관리 (하우저 연동)

  • [필수] 하우저 API 연동을 통한 실시간 주문 데이터 조회
  • [필수] 배송 상태 실시간 추적 (배송 준비 → 출고 → 배송 중 → 배송 완료)
  • [필수] 고객별 주문 이력 통합 조회
  • [필수] 배송 D-Day 카운트다운 표시 (참고 이미지 반영)
  • [선택] 주문 상태 변경 시 알림

3.2 고객 상담 이력 관리 (채널톡 연동)

  • [필수] 채널톡 Open API 연동을 통한 상담 내역 동기화
  • [필수] 고객별 상담 이력 타임라인 조회 (참고 이미지의 타임라인 UI 반영)
  • [필수] CS 담당자 상담 메모 작성 및 관리
  • [필수] 채널톡 웹훅을 통한 실시간 상담 알림 수신
  • [필수] 상담 분류 색상 코딩 — 불만/클레임(빨강), 일반문의(초록), 상담중(노랑) (참고 이미지 반영)
  • [선택] 고객에게 상담 결과 알림톡(SMS) 발송 (참고 이미지 반영)

3.3 관리자 및 사용자 관리

  • [필수] 사내 임직원 계정 생성/수정/삭제 (10~20명 규모)
  • [필수] 역할(Role) 기반 권한 관리 (슈퍼 관리자, CS 관리자, CS 상담사)
  • [필수] 이메일/비밀번호 기반 로그인
  • [선택] 비밀번호 초기화, 로그인 이력 관리

3.4 CRM 대시보드 및 통계

  • [필수] 상담사별 퍼포먼스 통계 (응대 건수, 처리 시간, 상태별 분류)
  • [필수] 통합 상담 이력 조회 — 날짜, 상태, 채널, 담당자 필터링 (참고 이미지 반영)
  • [필수] 고객 통합 검색 (이름, 연락처, 주문번호, 배송지)
  • [선택] VIP 고객 태깅 및 특이사항 경고 표시 (참고 이미지 반영)
  • [선택] 상담 이력 엑셀 다운로드 (참고 이미지 반영)

  • 기존 고객 데이터 이관 (별도 협의)
  • 다국어 지원
