제안 포인트

본 프로젝트의 핵심은 외부 시스템 연동(채널톡 + 하우저)과 CS 업무 최적화 UI입니다. 기존 B2B SaaS/어드민 패널 구축 경험을 바탕으로 다음과 같이 접근합니다:

  • 채널톡 Open API를 활용한 상담 이력 자동 동기화 및 실시간 웹훅 수신
  • 하우저 API 연동으로 주문/배송 상태 실시간 조회 및 고객 프로필에 통합 표시
  • RBAC(역할 기반 접근 제어)로 관리자/상담사 권한 분리 — EZ-Approve에서 7종 역할 관리 경험 보유
  • 상담사 퍼포먼스 대시보드 — 응대 건수, 평균 처리 시간, 만족도 등 핵심 지표 시각화
  • PC Web 기반으로 개발하여 별도 설치 없이 브라우저에서 즉시 사용 가능
  • 유사 프로젝트 코드 재활용 + AI 증강 개발(Google Jules, Copilot, Bolt 활용)로 90일 → 60일 단축

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

CS 인입 고객을 효율적으로 응대하고 관리하기 위해 새로운 전용 CRM 시스템을 구축합니다. 현재 채널톡과 배송업체(하우저) 시스템이 분리되어 있어 상담사가 고객 정보를 파악하기 위해 여러 시스템을 전환해야 하는 비효율이 발생하고 있습니다.

목적:

  • 채널톡 및 하우저 시스템 연동을 통한 고객 정보 통합 뷰 제공
  • 상담 이력과 주문/배송 상태를 한 화면에서 확인 가능한 환경 구축
  • 상담사 퍼포먼스를 한눈에 파악할 수 있는 대시보드 제공
  • 10~20명 규모의 사내 임직원이 사용할 수 있는 안정적인 시스템 구축

2. 사용자 정의

슈퍼 관리자시스템 전체 관리, 설정 변경 권한사용자 관리, 권한 설정, 시스템 설정, 전체 대시보드
CS 관리자CS 팀 리더, 상담사 관리상담사 퍼포먼스 조회, 상담 배정, 통계 대시보드

3. 기능 요구사항

3.1 주문 정보 관리 (하우저 연동)

  • [필수] 하우저 API 연동을 통한 실시간 주문 데이터 조회
  • [필수] 배송 상태 실시간 추적 (배송 준비 → 출고 → 배송 중 → 배송 완료)
  • [필수] 고객별 주문 이력 통합 조회
  • [선택] 주문 상태 변경 시 알림

3.2 고객 상담 이력 관리 (채널톡 연동)

  • [필수] 채널톡 Open API 연동을 통한 상담 내역 동기화
  • [필수] 고객별 상담 이력 타임라인 조회
  • [필수] CS 담당자 상담 메모 작성 및 관리
  • [필수] 채널톡 웹훅을 통한 실시간 상담 알림 수신
  • [선택] 상담 태그/카테고리 분류

3.3 관리자 및 사용자 관리

  • [필수] 사내 임직원 계정 생성/수정/삭제 (10~20명 규모)
  • [필수] 역할(Role) 기반 권한 관리 (슈퍼 관리자, CS 관리자, CS 상담사)
  • [필수] 이메일/비밀번호 기반 로그인
  • [선택] 비밀번호 초기화, 로그인 이력 관리

3.4 CRM 대시보드

  • [필수] 고객 360도 뷰 (상담 이력 + 주문 정보 통합)
  • [필수] 상담사 퍼포먼스 지표 (응대 건수, 평균 처리 시간)
  • [선택] 일별/주별/월별 통계 차트
  • [선택] 고객 검색 및 필터링

5. 기술 제약사항

  • 채널톡 Open API 호출 제한(Rate Limit)에 따른 폴링 간격 조정 필요
  • 하우저 API 문서 및 접근 권한은 클라이언트로부터 제공받아야 함
  • PC Web 기반 — 최신 브라우저 필수 (IE 미지원)
  • 서버 인프라는 클라이언트와 협의 후 결정 (AWS 권장)

6. 범위 외 사항 (Out of Scope)

  • 모바일 앱 개발 (PC Web 전용)
  • 채널톡 내 챗봇 개발 또는 자동 응답 시스템
  • 하우저 이외의 배송업체 연동
  • 결제 시스템 연동
  • AI 기반 자동 상담 분류/응답 추천
  • 대규모 트래픽(100명 이상) 대응을 위한 인프라 확장

EZ-Approve (전자결재 · 업무 관리 올인원 플랫폼)

B2B SaaS · 2026.01~2026.03 (9주) · 504 PR · 2,764 파일

프로젝트 개요

한국 기업의 분산된 결재·문서관리 프로세스를 통합하는 B2B SaaS 플랫폼. 엔터프라이즈 수준의 전자결재를 단일 대시보드에서 관리하고, 토큰 기반 셀프서비스 포털에서 외부 이해관계자의 등록·신청·총회 참석을 처리합니다. 50+ 페이지, 34 엔티티, 120~150 API 엔드포인트 규모입니다.

핵심 기능

  • 엔터프라이즈급 결재 엔진 — 8종 결재 행위(상신·합의·전결·선결·후결·대결)
  • RBAC 기반 7종 사용자 역할 관리 + CASL 권한 제어
  • 12개 외부 서비스 연동 (Firebase FCM, Ppurio SMS, NICE 본인인증, Gmail SMTP 등)
  • 170파일 Lexical 편집기 (26 커스텀 노드, 40+ 플러그인, Yjs 실시간 협업)
  • 6계층 보안 아키텍처 (JWT + 2FA + IP 화이트리스트 + Brute Force 방지 + Rate Limiting + Helmet)
  • AI 증강 개발 — Google Jules, Copilot, Bolt 활용

본 프로젝트와의 유사점

B2B 관리자 대시보드 구조, RBAC 7종 역할 관리, 12개 외부 API 연동 패턴이 CS CRM 시스템과 직접 연결됩니다. 특히 다종 사용자 역할 관리와 실시간 데이터 조회 대시보드 구축 경험이 본 프로젝트에 바로 활용 가능합니다.

기술 스택

Harmony Link (노인 주간보호센터 통합 관리 플랫폼)

멀티플랫폼 관리 시스템 · 2025 (~6개월) · 133K+ LOC · 6개 플랫폼

프로젝트 개요

노인 주간보호센터의 돌봄 기록, 투약 관리, 보호자 소통, AI 건강분석을 하나로 통합한 멀티플랫폼 관리 시스템. 6개 플랫폼(Android, iOS, Web, macOS, Windows, Linux)을 지원하며 140+ API 엔드포인트, 90개 DB 마이그레이션, 2,532개 다국어 키를 갖추었습니다.

핵심 기능

  • 이용자(고객) 이력 관리 — 보호일지, 식단, 사진앨범, 일정, 출석 디지털화
  • 실시간 푸시 알림 시스템 (Firebase FCM) + 보호자 소통
  • RBAC 6개 역할 기반 권한 관리 + CASL
  • 관리자 웹 패널 (Next.js 13 + MUI)
  • 외부 서비스 연동 (Firebase Auth/FCM/Crashlytics, OpenAI API)
  • AWS CDK 기반 완전 자동화 인프라 + 8개 CI/CD 워크플로우

본 프로젝트와의 유사점

고객(이용자) 이력 통합 관리, 실시간 알림, 다수 사용자 역할 관리가 CS CRM 시스템의 상담 이력 관리 및 사용자 권한 체계와 직접 유사합니다. 외부 API 연동(Firebase, OpenAI) 경험도 채널톡/하우저 연동에 활용 가능합니다.

기술 스택

Series B (투자조합 관리 · 업무 자동화 올인원 플랫폼)

올인원 관리 플랫폼 · 2023.11~2024.12 (14개월) · 1,652 PR · 80+ 엔티티

프로젝트 개요

한국 벤처캐피탈(VC) 업계에서 투자조합 관리, 전자결재, 규제 보고가 파편화된 문제를 해결하기 위한 올인원 SaaS 플랫폼. GP(운용사)는 결재·펀드관리·회계·VICS 규제보고를 단일 시스템에서 처리하고, LP(출자자)는 전용 대시보드로 펀드 현황을 실시간 조회합니다. 50+ 페이지, 80+ 엔티티, 179개 DB 마이그레이션, 22개 백엔드 도메인 모듈 규모입니다.

핵심 기능

  • AI 투자심사보고서 자동 생성 (ChatGPT API 연동)
  • GP/LP 펀드 라이프사이클 관리 — 펀드 등록~출자금~투자실행~성과보고
  • 한국형 전자결재 엔진 (합의·대결·선결·전결·부재위임)
  • VICS 규제 보고 자동화 — 5종 양식 + SFTP 자동 제출
  • 실시간 공동 편집 (Lexical + CRDT)
  • 5중 보안 아키텍처 (JWT + 2FA + NICE 본인인증 + IP 화이트리스트 + GeoIP)

본 프로젝트와의 유사점

대규모 데이터 통합 조회 대시보드(80+ 엔티티), 다중 외부 API 연동(Firebase, SMS, NICE, ChatGPT) 경험이 CRM의 고객 360도 뷰 구축에 직접 활용 가능합니다. 179개 마이그레이션 규모의 복잡한 데이터 모델링 경험은 안정적인 DB 설계를 보장합니다.
